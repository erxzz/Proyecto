El proyecto implementa un sistema orientado a objetos para la administración de un centro de adopción de mascotas. Contiene clases para representar mascotas, personas (adoptantes y trabajadores), adopciones y el propio centro. El objetivo es permitir el registro, administración y control básico de procesos de adopción.

El sistema ofrece funcionalidades como:

1. Ingreso y salida de mascotas del centro.
2. Registro de adoptantes.
3. Confirmacion y cancelacion de adopciones
4. Control de estados de las mascotas.
5. Consulta de mascotas disponibles.

Las instrucciones de ejecucion:
1. Cada elemento como mascota o persona se debe asignar como un objeto de una clase
2. De igual manera, los metodos de adopción tambien estan como clases

Decisiones de Diseño

1. Uso de herencia desde clases basicas a mas especificas
2. Uso de listas para facilitar el contenido de las variables
3. Uso de isinstance para validar el tipo de clase
4. Uso de comprension de listas

Limitaciones
1. Falta de implementacion de uso en la clase de trabajador
2. Falta de implementacion de metodos de comparacion
